# 通勤助手使用指南

## 功能概述

智能助理现已集成高德地图通勤助手功能，可以智能识别用户的通勤相关需求并提供以下服务：

### 核心功能
1. **POI 搜索** - 搜索地点、兴趣点
2. **路线规划** - 驾车、步行、公交路线
3. **附近查询** - 查找周边设施
4. **连续搜索** - 沿途设施查询

## 使用示例

### 1. 搜索地点
```
用户: 搜索附近的咖啡店
用户: 找加油站
用户: 北京三里屯的餐厅在哪里
```

### 2. 路线规划
```
用户: 从北京站到天安门的路线
用户: 从家到公司怎么走
用户: A到B的路线规划
```

### 3. 公交查询
```
用户: 从北京站到天安门的公交路线
用户: 公交从家到公司
```

### 4. 驾车导航
```
用户: 驾车从北京到上海
用户: 开车从家到机场怎么走
```

### 5. 步行路线
```
用户: 步行从北京站到天安门
用户: 走路从家到地铁站
```

### 6. 附近查询
```
用户: 附近的餐厅
用户: 附近哪里有停车场
```

## 技术实现

### 前端（Flutter）
- 在 `ai_gateway.dart` 中添加了 `_isCommuteRequest()` 方法
- 智能检测通勤相关关键词，直接将请求转发到后端

### 后端（Python Flask）
- 新增 `commute_helper.py` 模块
- 集成高德地图 REST API
- 支持多种路线规划模式

## 工作流程

1. 用户输入通勤相关请求
2. Flutter 前端检测关键词
3. 请求转发到 Python 后端
4. 后端调用高德地图 API
5. 返回格式化的结果给用户

## 连续搜索功能

通勤助手支持连续搜索模式，用户可以：
- 询问沿途设施（如"沿途搜索加油站"）
- 查找路上休息区
- 规划多点路线

## 注意事项

1. **网络要求**：需要稳定的网络连接访问高德地图 API
2. **API 密钥**：确保高德地图 API 密钥有效且有足够配额
3. **地址格式**：支持自然语言地址，但详细地址更准确
4. **服务限制**：高德地图 API 有调用频率限制

## 测试方法

1. 启动后端服务：
```bash
cd backend
python index.py
```

2. 启动 Flutter 应用

3. 输入测试命令：
```
"从北京站到天安门的路线"
"搜索附近的咖啡店"
"附近的餐厅"
```

## 故障排除

### 常见问题
1. **连接失败**：检查后端服务是否启动
2. **API 错误**：验证高德地图 API 密钥是否有效
3. **无结果**：尝试使用更通用的关键词

### 调试信息
后端会输出详细的 API 调用日志，便于排查问题。
